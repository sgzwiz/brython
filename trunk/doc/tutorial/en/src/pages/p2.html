HTML

Let's start by looking at some HTML.

<html> <body> Page text here. </body> </html>

If you've never seen HTML before, it's fairly simple. Each part with an angle bracket is called a tag. The first of each pair is called an opening tag. The second (with a /) is called a closing tag. If you copy that into your editor, and save it as name.html, you could open it with your browser. Let us do so. (If you got a 404 error, make sure the server is running!).

What is happening here? Nothing too complicated. The

<html> tag tells the browser that this is an HTML document. The <body> tag contains most of the "stuff" that goes into a web page. But we can do better than that. Let's try:

<html> <body style="background-color:green"> Page text here. </body> </html>

I'm sure you can figure out what that does, and here is your confirmation.

Adding a bit more depth to it, we come to something like a complete webpage.

<html> <head> <title> This title appears at the top of the page! </title> </head> <body> <table border="1"> <tr> <th>The First Header</th> <th>The Second Header</th> <th>Third Cell Header</th> </tr> <tr> <td>Under one</td> <td>Under two</td> <td>Under three</td> </tr> <tr> <td>Another cell</td> <td>Words</td> <td>I refuse to put foo</td> </tr> </table> </body> </html> Once again, it isn't too difficult to see what's happening here. The head section contains information about the title which is displayed in the title bar or tab of the browser. The body contains a table. The table in turn contains a few tr tags, which stands for "Table Row". Each row contains a <th> or <td> which are Table Headers and Table Data respectively. The Table Headers are the bolded elements. The Table Data are the normal cells. Inside the table tag <table border="1"> the cell boarders are set to a width of one pixel.

At this point I encourage you to copy some of these files over to the playground folder. Try and make the table 4x4 instead of 3x3. What happens if a cell is empty? What happens if you miss a tag?
